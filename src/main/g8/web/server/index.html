OAuth 2.0 - Authorization Server

The authorization server acts as an intermediary between the client application and the resource owner. It provides the following endpoints:

  • /authorize

    This is where the client application redirects the resources owner to. This endpoint expects the following parameters:

    • response_type

      Its (mandatory) value is code when the client hopes for the authorization code grant.

    • client_id

      Its (mandatory) value is the unique client identifier provided by the authorization server at registration,

    • redirect_uri

      It is the (optional) URI the client wants to be redirected to in case the resource owner approves.

    Requests to this endpoint are called authorization requests and they look like as follows

    GET /authorize?response_type=code&client_id=my-awsome-client&redirect_uri=https%3A%2F%2Fclient%2Enet%2Fcallback HTTP/1.1
Host: oauth2.server.net
  • /approve
  • /deny
